Building a Home Server with Raspberry Pi 3B

Published: June 10, 2024 (Updated: 2026)

The Pivot: Why I Switched

I started my home lab journey repurposing an old Ideapad laptop. It was free and had a built-in battery (UPS). However, it had downsides: it was bulky, ran hot, and the fan noise was noticeable at night.

I decided to migrate to a Raspberry Pi 3B. It's completely silent, fits in the palm of my hand, and sips power (< 4W at idle). It’s the perfect "set and forget" device for a 24/7 home server.

The Hardware

You don't need much to get started. Here is my minimalist setup:

  • Raspberry Pi 3B: Affordable and powerful enough for lightweight containers.
  • 16GB SanDisk Ultra MicroSD: For the OS and containers.
  • 5V 2.5A Power Supply: Stability is key; under-voltage causes weird crashes.
  • Ethernet Cable: Wi-Fi is fine, but wires are forever.

The Setup: Headless & Clean

I used the Raspberry Pi Imager to flash Raspberry Pi OS Lite (64-bit). The "Lite" version has no desktop environment, saving precious RAM for our services.

Pro Tip: Click the "Settings" gear icon in the Imager to:

  • Enable SSH (Authentication with public key).
  • Set the hostname (e.g., pi-server).
  • Configure Wi-Fi (as a backup to Ethernet).

The Stack: Docker is King

Gone are the days of installing services directly on the OS. I use Docker to keep everything isolated and clean. If a service breaks, I just kill the container, not the server.

Installing Docker on a Pi is a one-liner:

curl -sSL https://get.docker.com | sh
sudo usermod -aG docker $USER

Service 1: Pi-hole (Network Ad-Blocker)

The first thing running on my Pi is Pi-hole. It acts as a DNS sinkhole, blocking ads and trackers for every device on my Wi-Fi (phones, tablets, smart TV).

I run it via Docker Compose:

version: "3"
services:
  pihole:
    container_name: pihole
    image: pihole/pihole:latest
    ports:
      - "53:53/tcp"
      - "53:53/udp"
      - "80:80/tcp"
    environment:
      TZ: 'Asia/Kolkata'
    volumes:
      - './etc-pihole/:/etc/pihole/'
      - './etc-dnsmasq.d/:/etc/dnsmasq.d/'
    restart: unless-stopped

Service 2: Samba (Simple NAS)

I attached a 1TB external HDD to store backups and media. Samba makes this drive accessible to my Windows PC and Mac over the network. 

sudo apt install samba samba-common-bin
# Edit /etc/samba/smb.conf to add your drive sharing settings

Service 3: Cloudflare DDNS (Dynamic IP)

Since my ISP assigns a dynamic IP address, accessing my server remotely can be tricky if the IP changes. I use Cloudflare as my DNS provider and a custom script to automatically update my domain's A-record whenever my home IP changes.

I wrote a custom script for this which is available on my GitHub. It fetches my public IP and uses the Cloudflare API to update the record.

Setup
# Clone the script
git clone https://github.com/abhijithabhiakl/cloudflare-ddns-script
cd cloudflare-ddns-script
chmod +x update_ip.sh

# Add to crontab to run every 5 minutes
crontab -e
# Add line: */5 * * * * /home/pi/cloudflare-ddns-script/update_ip.sh

Conclusion

Migrating to the Raspberry Pi 3B was the best decision for my home lab. It's silent, efficient, and running Docker makes managing services a breeze. Checkout my other blogs to see more about my home lab.
